Themes & Styling

Tema & Styling

shadcn is built on CSS variables. Every component uses tokens like --primary, --background, --border. Switching an entire visual theme = one class on <body>.

shadcn dibangun di atas CSS variable. Setiap component pakai token seperti --primary, --background, --border. Ganti seluruh tema visual = satu class di <body>.

oklch cheat sheet

Cheat sheet oklch

oklch( L C H )
oklch( Lightness Chroma Hue ) 0 → 1 0 → .4 0 → 360° L: 0.97 = near-white | 0.50 = mid | 0.12 = near-black C: 0.00 = grey | 0.08 = pastel | 0.22 = vivid H: 0=red 40=orange 60=yellow 140=green 220=blue 350=pink Background → L 0.96+, C 0.01–0.02 Primary → L 0.55–0.70, C 0.18–0.26 Border → same hue, L 0.85–0.92

Bugs & Fixes

Bug & Perbaikan

Known errors from real sessions. For any error, paste it directly into Claude Code with "Fix this error" — resolves most cases automatically.

Error yang diketahui dari sesi nyata. Untuk error apapun, paste langsung ke Claude Code dengan "Fix this error" — sebagian besar langsung terselesaikan.

Bug 01 EPERM: operation not permitted, scandir ~/.Trash
EPERM: operation not permitted, scandir '/Users/yourname/.Trash'

Cause: Running shadcn add from your home directory ~. The CLI scans the current directory tree and hits protected macOS system folders.

Penyebab: Menjalankan shadcn add dari home directory ~. CLI scan direktori saat ini dan menabrak folder sistem macOS yang diproteksi.

Fix — cd into project first Fix — masuk ke folder project dulu
yourname@Mac ~ % npx shadcn@latest add button ← ❌ home dir # Fix: cd into your project folder cd /path/to/your-project npx shadcn@latest add button -c apps/web ← ✓ # Sanity check: ls should show package.json
Bug 02 zsh: no matches found: [component]
zsh: no matches found: [component] zsh: command not found: -c

Cause: [component] is a placeholder, not a literal value. zsh interprets square brackets as a glob pattern. Replace it with the actual component name.

Penyebab: [component] adalah placeholder, bukan nilai literal. zsh menginterpretasikan kurung kotak sebagai pola glob. Ganti dengan nama component aslinya.

npx shadcn@latest add [component] -c apps/web ← ❌ placeholder npx shadcn@latest add button -c apps/web ← ✓ npx shadcn@latest add --all -c apps/web ← install everything
Bug 03 Missing 'packageManager' — Turborepo won't start Missing 'packageManager' — Turborepo tidak mau start
× Could not resolve workspaces. ╰─▶ Missing packageManager field in package.json

Cause: Turborepo 2.x requires the root package.json to declare which package manager you use.

Penyebab: Turborepo 2.x mengharuskan root package.json mendeklarasikan package manager yang digunakan.

root package.json
{ "name": "my-monorepo", "private": true, "packageManager": "npm@10.8.2", ← add this "scripts": { ... } } # Find your version: npm --version
Bug 04 Component added to wrong folder in monorepo Component masuk folder yang salah di monorepo
It looks like you are running add [component] from a monorepo root. Use the -c flag to target a specific workspace.
It looks like you are running add [component] from a monorepo root. Use the -c flag to target a specific workspace.
npx shadcn@latest add button -c apps/web ← app-specific npx shadcn@latest add button -c packages/ui ← shared across apps npx shadcn@latest add --all -c packages/ui ← install everything shared
Bug 05 Module not found: @/components/ui/accordion
Module not found: Can't resolve '@/components/ui/accordion' Import map: aliased to './components/ui/accordion' inside [project]/apps/web

Cause: Components are in packages/ui but imported via @/components/ui/ which only works when components are in the same app. Use the shared package name.

Penyebab: Component ada di packages/ui tapi di-import lewat @/components/ui/ yang hanya bekerja di app yang sama. Gunakan nama shared package.

# Find your package name cat packages/ui/package.json | grep '"name"' → "name": "@workspace/ui" import { Button } from "@/components/ui/button" ← ❌ import { Button } from "@workspace/ui/components/button" ← ✓ # Fix all files: Cmd+Shift+H in VS Code # search: @/components/ui/ # replace: @workspace/ui/components/
Bug 06 Localhost not opening / port conflict Localhost tidak terbuka / konflik port
⚠ Port 3000 is in use, trying 3001 instead.
⚠ Port 3000 is in use, trying 3001 instead.

Cause: Port 3000 is occupied. Next.js auto-bumps to the next available port — always read the terminal output for the actual URL.

Penyebab: Port 3000 sudah terpakai. Next.js otomatis beralih ke port berikutnya — selalu baca output terminal untuk URL yang sebenarnya.

# Follow what the terminal says — open :3001 if it shows :3001 # Force a specific port npm run dev -- --port 4000 # Kill what's on port 3000 lsof -ti:3000 | xargs kill -9 # Monorepo: start only one app cd apps/web && npm run dev npx turbo dev ← starts all apps in parallel
